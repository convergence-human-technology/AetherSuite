# Security Policy

## Supported versions

AetherSuite is provided as-is. There is no official support contract, no paid tier, and no dedicated security team behind this project.

The latest version available in this repository is the one that receives attention. Older versions are not maintained.

## Reporting a security issue

If you find a security vulnerability, please do not open a public issue. A public issue would expose the problem to everyone before it can be fixed.

Instead, send a private message through GitHub. Go to the Security tab of this repository and use the "Report a vulnerability" button. Describe the issue in as much detail as you can, including steps to reproduce it.

We will acknowledge your report and assess the issue. There is no guaranteed response time, as this is a volunteer-maintained project.

## Your responsibility as a user

Because AetherSuite is open source and freely distributed, the responsibility for keeping it secure on your device belongs to you.

This means :

- Checking this repository regularly for updates and new releases.
- Rebuilding and reinstalling the apps when a new version is available.
- Deciding for yourself whether the current version meets your security needs.

We provide the source code transparently so that anyone can audit it. What you do with it is your responsibility.
